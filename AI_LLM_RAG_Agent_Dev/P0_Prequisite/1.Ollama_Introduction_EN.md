## What is Ollama?

[Ollama](https://ollama.com) is an open-source tool that lets you run large language models (LLMs) on your local machine. With zero configuration, you can download, run, and manage open-source models using a single command—all while keeping your data private and local.

### Why Ollama?

- **Zero Configuration** — No need to mess with CUDA, Python environments, or dependencies. Works out of the box.
- **Privacy First** — All data stays on your machine, nothing sent to the cloud.
- **Hardware Friendly** — Automatically detects and uses GPU; falls back to CPU if no GPU available.
- **Rich Ecosystem** — Supports 100+ open-source models, from lightweight to extra-large.

### macOS

```bash
brew install ollama
```

Or download directly from the [official website](https://ollama.com/download).

### Linux

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

### Windows

Download the installer from the [official website](https://ollama.com/download).

### Docker

```bash
docker pull ollama/ollama
```

## Quick Start

### 1. Start the Service

```bash
ollama serve
```

The service runs at `http://localhost:11434` by default.

### 2. Run Your First Model

```bash
ollama run llama3.2
```

The model will be downloaded automatically on first run (Llama 3.2 is about 2GB). Once downloaded, you'll enter an interactive chat session:

```
>>> Hello!
Hi there! How can I help you today?

>>> /bye
```

Type `/bye` to exit the conversation.

### 3. Common Commands Cheat Sheet

| Command               | Description                         |
| --------------------- | ----------------------------------- |
| `ollama list`         | List downloaded models              |
| `ollama pull <model>` | Download a model without running it |
| `ollama run <model>`  | Download and run a model            |
| `ollama rm <model>`   | Remove a model                      |
| `ollama ps`           | List currently running models       |
| `ollama stop <model>` | Stop a running model                |

### Model Tags Explained

Models often have multiple versions called "tags":

```bash
ollama run llama3.2        # Default version (recommended)
ollama run llama3.2:1b     # Lightweight version, faster
ollama run llama3.2:3b     # Standard version
```

> **Memory Requirements**: 7B models need ~8GB RAM, 13B needs ~16GB, 33B needs ~32GB.

## API Usage

Once Ollama is running, it provides an HTTP API for integration:

### Chat API

```bash
curl http://localhost:11434/api/chat -d '{
  "model": "llama3.2",
  "messages": [{"role": "user", "content": "Hello!"}]
}'
```

### Generate API (Streaming)

```bash
curl http://localhost:11434/api/generate -d '{
  "model": "llama3.2",
  "prompt": "Write a poem about spring"
}'
```

### Python Example

```python
import requests

response = requests.post('http://localhost:11434/api/chat', json={
    'model': 'llama3.2',
    'messages': [{'role': 'user', 'content': 'Hello!'}],
    'stream': False
})
print(response.json()['message']['content'])
```

For more API details, see the [official API documentation](https://github.com/ollama/ollama/blob/main/docs/api.md).

## Advanced Features

### Custom Models with Modelfile

Create customized models using a Modelfile:

```dockerfile
FROM llama3.2

SYSTEM """You are a professional coding assistant specializing in Python and JavaScript."""

PARAMETER temperature 0.7
```

Build and run:

```bash
ollama create my-assistant -f Modelfile
ollama run my-assistant
```

### Multi-line Input

In interactive mode, use `"""` for multi-line text:

```
>>> """
... Line 1
... Line 2
... Line 3
... """
```

## Troubleshooting

**Q: How do I completely stop Ollama?**

Type `/bye` in interactive mode, or press `Ctrl+C` in the terminal. To stop the background service:

```bash
# macOS/Linux
pkill ollama

# Windows PowerShell
Get-Process ollama | Stop-Process
```

**Q: Where are models stored?**

- macOS: `~/.ollama/models/`
- Linux: `/usr/share/ollama/.ollama/models/`
- Windows: `C:\Users\<username>\.ollama\models\`

**Q: How do I force CPU/GPU usage?**

Ollama uses GPU by default. To force CPU:

```bash
OLLAMA_GPU_OVERHEAD=1 ollama run llama3.2
```
